üîß Compilation vs. Interpretation

One of the biggest differences between Java and C++ is how your code becomes executable.

Aspect Java C++
Compilation Compiled to bytecode (.class files) Compiled directly to machine code (.exe or binary)
Execution Runs on Java Virtual Machine (JVM) Runs directly on the operating system
Portability Write once, run anywhere Must recompile for each platform
Speed Slightly slower due to JVM overhead Generally faster (native execution)
Memory Management Automatic garbage collection Manual memory management (you control it!)

üí° Why This Matters

In C++, you have more control but also more responsibility. You'll see this especially with memory management in Week 5!

üëã Hello World: Java vs. C++

Let's compare the classic "Hello World" program in both languages:

Java Version

Java 
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

C++ Version

C++ 
#include <iostream>
using namespace std;

int main() {
    cout << "Hello, World!" << endl;
    return 0;
}

üìä Data Types

Most basic data types are similar, but there are some important differences:

Type Java C++ Notes
Integer int (32-bit) int (size varies!) C++ int is platform-dependent
Floating point double double Same in both
Character char (16-bit Unicode) char (8-bit ASCII) Different sizes!
Boolean boolean bool Different keyword
String String (class) string (class) Lowercase 's' in C++

‚ö†Ô∏è Common Gotcha: Boolean Values

In Java: true and false

In C++: true and false (same!)

BUT: C++ also treats 0 as false and any non-zero value as true

üéØ Key Takeaways

  • C++ compiles directly to machine code (faster but less portable than Java)
  • Use cout << for output and cin >> for input
  • Header files (#include) bring in library functionality
  • using namespace std; lets us use cout instead of std::cout
  • Manual memory management gives you power and responsibility
  • The main() function returns an int (0 = success)
