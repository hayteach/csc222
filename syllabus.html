üìñ COURSE DESCRIPTION (The Fun Stuff!)

Ready to level up your coding game? üéÆ This course is where the magic happens! We'll explore the art and science of data structures and algorithms - think of them as the LEGO blocks of programming. You'll learn how to build elegant, efficient programs that would make even experienced developers nod in approval. Warning: You may develop an unhealthy obsession with Big O notation. üòÖ

üéØ Course Objectives/Learning Outcomes (aka What You'll Be Able to Brag About):

Course Objectives:

  • üìä Write programs that use each of the following data structures: arrays, records, strings, linked-lists, stacks, queues and hash tables (collect 'em all!).
  • üîÑ Implement, test and debug simple recursive functions and procedures (inception for code!).
  • ‚öñÔ∏è Evaluate tradeoffs in lifetime management (reference counting vs garbage collection) - it's like choosing between pizza toppings, but for memory!
  • üß© Explain how abstraction mechanisms support the creation of reusable software components (DRY - Don't Repeat Yourself!).
  • üé® Design, implement, test, and debug simple programs in an object-oriented programming language.
  • ü§î Compare and contrast object-oriented analysis and design with structured analysis and design.

Learning Outcomes:

Upon successful completion of the course the student should be able to:

  • Write programs that implement and use each of the following data structures: arrays, records, strings, linked lists, stacks, queues, and hash tables.
  • Compare iterative and recursive solutions for elementary problems such as factorial.
  • Compare and contrast object-oriented analysis and design with structured analysis and design.
  • Discuss the representation and use of primitive data types and built-in data structures.
  • Choose the appropriate data structure for modeling a given problem.
  • Identify and describe the properties of a variable such as its associated address, value, scope, persistence, and size.
  • Describe how the class mechanism supports encapsulation and information hiding.
NOTE: You will see these learning outcomes at times in the grading rubrics for assignments. When you see one, it means that I feel that all or part of this assignment allows you to demonstrate your ability to meet this outcome.

ü§ñ AI Usage Policy (Yes, We're Living in the Future!)

So, AI is a thing now, right? Tools like GitHub Copilot and ChatGPT are like having a really smart (but sometimes quirky) coding buddy. You CAN use them in this course - but with great power comes great responsibility! Think of AI as your study partner, not your replacement brain. üß†‚ú®

Permitted Uses of AI:

  • Learning and Understanding: Use AI to help explain concepts, debug code, or understand error messages.
  • Code Review: Use AI to review your code for best practices, optimization suggestions, or potential bugs.
  • Brainstorming: Use AI to explore different approaches to solving a problem or implementing a data structure.
  • Documentation: Use AI to help generate or improve code comments and documentation.

Required Practices:

  • Understanding: You must understand any code that you submit. If asked to explain your code, you should be able to do so clearly.
  • Attribution: When AI generates significant portions of your code, include a comment indicating this (e.g., "// AI-assisted implementation").
  • Testing: All code, whether AI-generated or not, must be tested and verified by you to ensure it works correctly.
  • Learning Focus: AI should supplement your learning, not replace it. Use AI as a tutor, not as a solution provider.

Prohibited Uses:

  • Exams: AI tools may not be used during examinations unless explicitly permitted.
  • Blind Copying: Copying AI-generated code without understanding it is considered academic dishonesty.
  • Misrepresentation: Submitting AI-generated work as entirely your own without attribution is a violation of academic integrity.
IMPORTANT: The goal of this course is to develop your problem-solving skills and understanding of data structures and algorithms. While AI can be a valuable tool, over-reliance on it will prevent you from achieving the learning outcomes of this course. Remember: in interviews and professional settings, you'll need to demonstrate your own knowledge and skills.
