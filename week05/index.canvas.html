In the real world, we have occasions when we accumulate stuff into a holding area for later use. Each of these areas can have different properties. A Stack ADT (abstract data type) acts just like what it is ... it is a stack. So think of a stack of papers or a stack of DVDs or a stack of plates. If you think about it, the first thing you can get at is actually the last thing you added to the stack. So a stack is knows as a LIFO (Last In - First Out) structure. 

Last plate on the stack is the first one to be used!

  

 

Layout Table

For the Stack ADT, we want to create an interface so that all stacks, regardless of how we implement them, will have and use a common set of methods. In essences, from a user standpoint, it won't matter whether we implement the stack using an array or a linked-list. Their program will work with  either since all of the methods are named the same and produce identical output.

This is, by the way, one of the primary reasons to use interfaces for common ADTs.

ADT are virtual and abstract. Because of this, there can be no corresponding implementation file hence there is no include statement for stackADT.cpp because that file will not exist. The actual implementation details will be coded in any class that implements this interface.

Now that we've defined how a stack works, we can look at creating one that we can use in other programs to solve problems. The first implementation we will look at is that of using an Array to store the elements we put onto the stack. First, we will look at the UML diagram for an array-based stack that implements all of the stack operations from the stackADT interface plus the additional attributes and methods we need as described below.

 

Layout Table

 

  In addition to the interface methods of:

  • initializeStack()
  • isEmptyStack()
  • siFullStack()
  • push(ItemType)
  • top()
  • pop()

We need the following attributes to use an array:

  • maxStackSize - to define how big our array is (or the maximum number of ItemType things we can store in the stack
  • stackTop - the current top of the stack which is necessary when the stack is only partially filled
  • *list - pointer to our array of ItemType things

We also have the added methods so that we can construct arrayStack objects:

  • arrayStack(int = 100) - default arrayStack constructor
  • arrayStack(arrayStack) - constructor for creating a stack from an existing stack
  • copyStack(arrayStack) - copies an existing arrayStack
  • ~arrayStack() - destructor for the arrayStack object
  • operator=  - define what the assignment operator, =, will do when using arrayStack objects
