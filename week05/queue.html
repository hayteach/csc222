In the real world, we have occasions when we accumulate stuff into a holding area for later use. Each of these areas can have different properties. A Queue ADT (abstract data type) acts just like a line at the DMV or a Bank. If you think about how these lines work, the first thing you can get at is the first thing you added to the queue. So a queue is known as a FIFO (First In - First Out) structure. 

First person in line is the first person to get service!

  

 

Layout Table

Just like for the Stack ADT, we want to create a Queue interface so that all queues, regardless of how we implement them, will have and use a common set of methods. In essences, from a user standpoint, it won't matter whether we implement the queue using an array or a linked-list. Their program will work with either since all of the methods are named the same and produce identical output.

This is, by the way, one of the primary reasons to use interfaces for common ADTs.

ADT are virtual and abstract. Because of this, there can be no corresponding implementation file hence there is no include statement for queueADT.cpp because that file will not exist. The actual implementation details will be coded in any class that implements this interface.

Now that we've defined how a queue works, we can look at creating one that we can use in other programs to solve problems. The first implementation we will look at is that of using an Array to store the elements we put into our queue. First, we will look at the UML diagram for an array-based queue that implements all of the queue operations from the queueADT interface plus the additional attributes and methods we need as described below.

 

Layout Table

 

  In addition to the interface methods of:

  • initializeQueue()
  • isEmptyQueue()
  • isFullQueue()
  • front()
  • rear()
  • addQueue(const ItemType&)
  • deleteQueue()

We need the following attributes to use an array:

  • maxQueueSize - to define how big our array is (or the maximum number of ItemType things we can store in the stack
  • count - the current size of the Queue (how many things are currently in the queue
  • queueFront - the current front of the queue which is necessary when you want to take things out of the queue
  • queueRear - the current rear of the queue which is necessary when you want to add things to the queue
  • *list - pointer to our array of ItemType things

We also have the added methods so that we can construct arrayQueue objects:

  • arrayQueuu(int = 100) - default arrayQueue constructor
  • arrayQueue(arrayQueue) - constructor for creating a stack from an existing queue
  • ~arrayQueue() - destructor for the arrayQueue object
  • operator=  - define what the assignment operator, =, will do when using arrayQueue objects
