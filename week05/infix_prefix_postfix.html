Infix, prefix and postfix are 3 different ways to write arithmetic expressions.

infix - this is the way we learned in elementary school

prefix - also called Polish Notation after the Polish mathemetician that developed it, is a way to write the same mathematical expression by putting the operator before the value so parentheses are not needed

postfix - also called Reverse Polish Notation (RPN) because it is like prefix except the operator comes after the value also not requiring parentheses.

Here are some examples:

So ... why is this important you may ask....

Many compilers convert arithmetic expressions into some form of Postfix notation. This is done because postfix notation can be easily evaluated by using a stack.

Consider this "infix" expression: (6 + 3) * 2

We can convert this to "postfix" as: 6 3 + 2 *
